This is a summary schema to import all related application schemas for GeoSciML v 3.2.0<br /></p><p>     <br /></p><p>GeoSciML is a data transfer standard that specifies a set of feature-types and supporting structures for information used in the solid-earth geosciences.  <br /></p><p>GeoSciML is scoped approximately to the information required to construct geologic maps, but also geological sampling, observation and analyses. "Map" is interpreted broadly, <br /></p><p>and not constrained to conventional 2-D semi-horizontal cartography, so the GeoSciML information structures also support representation of the distribution of <br /></p><p>geologic features on sections, along curvilinear-traverses and boreholes, and within volumetric samples, etc.  GeoSciML is primarily concerned with "interpreted" <br /></p><p>geology (units, structures, etc), but links to external schemas for the descriptions of observational data. <br /></p><p>      <br /></p><p>GeoSciML is factored into a set of 13 application schema sub-packages.  From v3.0 onwards, the component packages are maintained separately. <br /></p><p>      <br /></p><p>---<br /></p><p>      <br /></p><p>GeoSciML was developed under the auspices of the Interoperability Working Group (https://www.seegrid.csiro.au/twiki/bin/view/CGIModel/InteroperabilityWG/) <br /></p><p>of the Commission for the Management and Application of Geoscience Information (http://www.cgi-iugs.org), a commission of the International Union of Geological <br /></p><p>Sciences (http://www.iugs.org).  For the latest release, please see http://www.geosciml.org. <br /></p><p><br /></p><p>Copyright (c) Commission for the Management and Application of Geoscience Information 2013.  All rights reserved. <br /></p><p><br /></p><p>=========================================================================================<br /></p><p><br /></p><p>Changes from GeoSciML v3.0 to v3.1<br /></p><p><br /></p><p>- GeoSciML-Core package - changed cardinality of "classifier" to 1..* (voidable) <br /></p><p>- Geomorphology package - added "activity" (0..1) to NaturalGeomphologicFeature<br /></p><p>- The Borehole, Collection, EarthMaterial, GeologicAge, GeologicStructure, GeologicUnit, and LaboratoryAnalysis-Specimen schemas were advanced to v3.1 as they<br /></p><p>   import GeoSciML-Core v3.1.<br /></p><p>   <br /></p><p>=========================================================================================<br /></p><p><br /></p><p>Changes from GeoSciML v3.1 to v3.2.0 - 24 June 2013<br /></p><p><br /></p><p>- Changed import dependencies to import all v3.2 schemas<br /></p><p>- Changes to other GeoSciML packages:<br /></p><p>   - Borehole - added intervalBegin and intervalEnd attributes to DrillingDetails class.<br /></p><p>              - added mappedIntervalBegin and mappedIntervalEnd to MappedInterval class.<br /></p><p>              - the addition of discrete begin and end attributes for downhole intervals is to allow data providers to overcome limitations of various WFS <br /></p><p>                implementations which do not allow delivery of 1D spatial objects from non-spatial databases, and to work around the inability of WFS filter <br /></p><p>                to query spatial objects of varying SRS in a single WFS service.<br /></p><p>   - Collection - added collectionType attribute to the GSML class.<br /></p><p>   - Utilities - added GSML_QuantityRange class as a child of swe:QuantityRange.  The new class has two discrete attributes (lowerValue and upperValue) to deliver<br /></p><p>                 a quantity range.  This works around the inability of WFS filter to query a number pair such as in swe:QuantityRange/value.<br /></p><p>   - GeologicUnit and EarthMaterial - these packages now import Utilities to gain access to the new GSML_QuantityRange element.<br /></p><p>                 <br /></p><p>=========================================================================================<br /></p>
